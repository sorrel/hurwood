About the site

Page last updated on the 10th of February 2005

This page describes the recent update of this site. The previous version of the this page, describing the design, and the decision to write it in XHTML can be found in the Archive on the menu.

     

Standards

I've tried again to ensure the site is XHTML 1.0 strict, and uses a style sheet that adheres to the standards. I also strongly recommend that anyone using Internet Explorer downloads Firefox and uses that instead.

For this version of the site, I have stopped trying to keep all graphical files down to the smallest possible size. Most people I know now have a broadband connection, and if this sample is representative of the population at large, there is no need to keep images belong 25k per file.

HTML Editors

The previous website had been written with the help of a HTML Editor called Homesite 4. The current version of Homesite is 5.5, but I was unable to use this for two reasons.

Firstly, upgrading or buying a new version of Homesite costs money, and I didn't want to spend money on what is essentially a glorified text editor with syntax highlighting. Secondly, I no longer had a PC, and Homesite is not available for the Apple Mac.

I don't like emacs, or other multi-purpose editors very much as I find them far too involved for straightforward tasks, as they have so many options and configuration files. I also looked at some commerical editors such as SEEdit, or Page Spinner.

In the end, though, I used Taco, a basic but very easy to use HTML text editor. It does exactly what I want, i.e. syntax highlighting, and it doesn't try to interere with the pages you're writing. The only downside is that it doesn't have any XHTML syntax checking, but the W3C Validator Page is able to do that.

For an FTP Program, I used RBrowser Lite, which does what it ought to.

CSS

I also looked for a CSS Editor. Previously I had used Top Style, but again this is not available for the Mac. This was more of a loss than Homesite, as it's a very good program.

I looked at replacing it with CSSEdit, or the free Morphon editor. In the end, though, I just used Taco again, even though it didn't have any CSS syntax highlighting.

The Style Sheets used on the site

I didn't want to make many changes to the style sheets used on the site. The first style sheet I didn't change at all, apart from the addition of some new animals for the page symbols. This was very easy, apart from the IE PNG problem, of which more later.

The second style sheet I changed slightly. I added the max-width and min-width tags, to get some more control over the look of the finished page. It's a pity that Internet Explorer 6 doesn't support them, so I ended up setting a width tag too. Hopefully I can put it back to auto when IE catches up with everything else. I also changed the colour of the <em> tagged text, if anyone can tell!

Browsers: Much Improved

The site was built and checked on Safari and Firefox on the Mac. I also tested it with the old version of Internet Explorer (5.0) on the Mac, Opera, and IE 5.5 and 6.0. running under Windows.

Generally I found the standards of the browsers to be much better than a few years ago, with standards applied in a much better way than previously. (A big hello to any Netscape 4 users out there!)

Apart from Internet Explorer

I can't begin to describe how annoying IE 6 is when trying to use XHTML and style sheets. Ignoring the way they tend to ignore the standards and head off into their own set of selectors, the lack of support for CSS2 items like max-width is very frustrating. And don't get me started about PNG transparency. Aaaaaarrgggh!

The solutions used on this site

I spent quite a while looking for a solution to the IE PNG transparency problem. Solutions I tried included the following sites:

  • http://www.koivi.com/ie-png-transparency/
  • http://homepage.ntlworld.com/bobosola/
  • http://webfx.eae.net/dhtml/pngbehavior/pngbehavior.html
  • http://www.alistapart.com/articles/pngopacity/

While some of the above sites were very good, I couldn't get the solution to work with the way I implement images within style-sheets. Finally, I did find a solution, the Young Pup website. The full solution url is: http://www.youngpup.net/2001/sleight.

I don't like spending my time trying to fix problems in browsers, and hope this is the last time I have spend time worrying about this kind of issue.

Work To Be Done

This site is made up of a number of older sites, going back to 1999. Most of the older sites don't adhere to the W3C standards, and only this one uses CSS.I'm not going to make the older sites adhere to the current standards.

The work to be done depends on the new version of IE, whenever it arrives. Hopefully, at that point there will be no need for any hacks, and standard compliant XHTML will "just work" on all browsers.
