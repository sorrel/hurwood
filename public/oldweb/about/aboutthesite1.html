About the site

Page last updated on the 1st of July 2003

This page describes the motivations behind this website, and links to a few places that those also writing web pages may find interesting.

Why am I here?

This site was created for two reasons:

  • To let people who don't see us very often have a glimpse of what we're doing.
  • To improve our web programming skills.

In particular, this site has been updated recently to take advantage of Cascading Style Sheets. While looking at them, I discovered that quite a few things had changed in the four years since I last read a book on HTML.

Main Information Sources

I'd been planning to use Cascading Style Sheets (usually shortened to CSS) for quite a while, but with so many web-standards to catch up on, knowing where to start presented a bit of a challenge.

After looking at a number of books on the subject I decided that the best source of information was the actual source of the information: The World Wide Web Consortium. The W3C website is the home of the specifications of the languages used in the making of webpages.

With so much information to make sense from, what is really needed is a guide page, which thankfully the W3C have provided at www.w3.org/2002/03/new-to-w3c.

My recommended W3C starting points:

  • Basic HTML Tutorial, without this the web just wouldn't be the same.
  • XHTML Specification Document, HTML now has an additional X-factor.
  • CSS Specification Document, reading the spec is the best tutorial around.
  • The CSS Box Model, understanding this section of the spec is crucial.

Once I felt I had a good understanding of CSS syntax, the real work began: putting the information to work in a practical way.

Using CSS

Trying to just list the number of sites offering hints and tips on using CSS is a mammoth task, and one I certainly won't attempt here. Listed below are a few of the sites I consulted; all were useful in some way, and have described techniques that I have adapted for this site.

Web design sites I found useful:

  • www.WebMasterWorld.com, the best web discussion forum.
  • A List Apart, an excellent site for friendly top tips.
  • CSS Column Layouts, there are many great websites showing how to setup columns, this being one.

Browsers

I've tried to make sure that this site functions correctly on all four modern browsers (Netscape, Internet Explorer, Opera and Mozilla). In addition to that, I also wanted to make sure that it worked well and was legible if either Javascript or Style Sheets were turned off.

The most annoying problem was to make sure the page was readable using Netscape Navigator 4,and that the CSS validated at the same time. On all the pages except this one I list the two style sheets both as "alternate stylesheet", which NN4 will ignore; on this page I have one described as just a "stylesheet" as the validator searches for a prefered stylesheet, which is written as just "stylesheet". I added some javascript to ensure that NN4 automatically uses a different (empty) style sheet, and doesn't pick the main one, which it wouldn't be able to handle.
